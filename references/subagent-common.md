# 서브에이전트 호출 공통 규칙

## 호출 방법

모든 Reviewer 호출은 Claude Code의 `Agent` 도구로 수행한다.

### 기본 호출 패턴

```
Agent({
  description: "Round N - Reviewer 검토",
  model: "{선택된 모델}",
  prompt: "{자율 생성된 프롬프트}"
})
```

### 인자

| 인자 | 필수 | 설명 |
|------|------|------|
| `description` | O | 간단한 작업 설명 (예: "Round 1 - 초안 검토") |
| `model` | O | 사용할 모델: `sonnet`, `opus`, `haiku` |
| `prompt` | O | 자율 생성된 상세 지시사항 + 검토 대상 내용 전체 |

### 모델 옵션

| 모델 | `model` 값 | 특성 |
|------|-----------|------|
| Claude Sonnet | `sonnet` | 균형 잡힌 성능·속도. 기본 추천 |
| Claude Opus | `opus` | 최고 추론 능력. 복잡한 분석용 |
| Claude Haiku | `haiku` | 빠르고 경량. 단순 검토용 |

## 프롬프트 구성

메인 Claude가 매 호출마다 프롬프트를 자율 생성한다. 프롬프트에 반드시 포함할 요소:

```
1. 역할 지정: "당신은 독립적인 Reviewer입니다. 메인 Claude의 초안을 비판적으로 검토합니다."
2. 주제 및 맥락 설명
3. 현재 라운드 번호와 모드
4. 검토 대상 내용 (초안/수정안 전문)
5. 이전 라운드 피드백 (2라운드 이상일 경우)
6. 모드별 톤 지시 (건설적 피드백 / 비판적 반론 등)
7. 웹검색 지시 (주제 유형에 따라)
8. 응답 형식 지시
```

### 독립성 보장

서브에이전트가 메인 Claude의 결론을 그대로 추종하지 않도록 다음 문구를 프롬프트에 포함:

> "메인 Claude와 독립적인 시각에서 검토하라. 동의하지 않는 부분은 명확히 반박하고, 누락된 관점이나 대안을 제시하라."

## 세션 관리 — 맥락 주입 방식

매 라운드마다 새 `Agent`를 호출하고, 이전 라운드의 맥락을 프롬프트에 구조화하여 포함한다.

### 맥락 주입 프롬프트 구조

```
# 역할
당신은 독립적인 Reviewer입니다. ...

# 이전 라운드 요약
## Round 1
- Main 초안 핵심: {요약 또는 전문}
- Reviewer 검토 핵심: {요약 또는 전문}
- Main 판단: {수용/반박 항목}

## Round 2
- ...

# 현재 라운드 검토 대상
{수정된 초안 또는 재반박 전문}

# 지시사항
{모드별 톤에 따른 검토 지시}
```

### 라운드별 맥락 압축 규칙

| 라운드 | 이전 라운드 포함 방식 | 현재 라운드 |
|--------|---------------------|------------|
| Round 1 | (이전 없음) | 초안 전문 |
| Round 2 | Round 1 전문 포함 | 수정안 전문 |
| Round 3+ | Round 1~(N-2) 핵심 요약만, Round (N-1) 전문 포함 | 수정안 전문 |

**압축 시 요약에 포함할 항목**: 각 라운드의 주요 쟁점, 합의된 사항, 미해결 사항. 세부 논증이나 예시는 생략.

### 예시: 3라운드 Adaptive 모드

```
Agent({
  description: "Round 3 - Reviewer 검토",
  model: "sonnet",
  prompt: "당신은 독립적인 Reviewer입니다. ...

# 이전 라운드 요약
## Round 1 (요약)
- Main 초안: REST API 기반 설계 제안
- Reviewer 검토: 실시간성 부족 지적, WebSocket 도입 권장
- Main 판단: WebSocket 부분 수용, 하이브리드 구조로 수정

## Round 2 (전문)
- Main 수정안: [수정안 전문...]
- Reviewer 검토: [검토 전문...]
- Main 판단: [판단 전문...]

# 현재 라운드 검토 대상
[Round 3 수정안 전문...]

# 지시사항
수정된 내용을 검토하고, 합의에 도달했는지 판단해주세요."
})
```

### 선택적 최적화: SendMessage (실험적)

> **주의**: 이 기능은 실험적이며, 향후 변경될 수 있다.

`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` 환경변수를 설정하면 `SendMessage`로 기존 서브에이전트 세션을 이어갈 수 있다. 이 경우 이전 맥락이 서브에이전트 내부에 자동 유지되므로 프롬프트에 별도 주입할 필요가 없다.

```
# settings.json에 추가
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

**이 방식은 기본이 아님.** 맥락 주입 방식이 모든 환경에서 안정적으로 작동하므로, SendMessage는 사용자가 명시적으로 설정한 경우에만 활용한다.

## 응답 처리

Agent 도구의 반환값은 서브에이전트의 응답 텍스트 그 자체이다.

### 메인 Claude의 후처리

1. 서브에이전트 응답을 해당 라운드의 파일로 저장 (예: `round1_2_reviewer_review.md`)
2. 응답 내용을 분석하여 판단(decision) 작성
3. 다음 라운드 진행 여부 결정

### 서브에이전트 호출 실패 시

Agent 도구 호출이 실패한 경우:
1. 1회 재시도
2. 재시도 실패 시 메인 Claude가 단독으로 해당 라운드를 수행
3. `collab_summary.md`에 실패 사실 기록
