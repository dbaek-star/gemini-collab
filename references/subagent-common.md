# 서브에이전트 호출 공통 규칙

## 호출 방법

모든 Reviewer 호출은 Claude Code의 `Agent` 도구로 수행한다.

### 기본 호출 패턴

```
Agent({
  description: "Round N - Reviewer 검토",
  model: "{선택된 모델}",
  prompt: "{자율 생성된 프롬프트}"
})
```

### 인자

| 인자 | 필수 | 설명 |
|------|------|------|
| `description` | O | 간단한 작업 설명 (예: "Round 1 - 초안 검토") |
| `model` | O | 사용할 모델: `sonnet`, `opus`, `haiku` |
| `prompt` | O | 자율 생성된 상세 지시사항 + 검토 대상 내용 전체 |

### 모델 옵션

| 모델 | `model` 값 | 특성 |
|------|-----------|------|
| Claude Sonnet | `sonnet` | 균형 잡힌 성능·속도. 기본 추천 |
| Claude Opus | `opus` | 최고 추론 능력. 복잡한 분석용 |
| Claude Haiku | `haiku` | 빠르고 경량. 단순 검토용 |

## 프롬프트 구성

메인 Claude가 매 호출마다 프롬프트를 자율 생성한다. 프롬프트에 반드시 포함할 요소:

```
1. 역할 지정: "당신은 독립적인 Reviewer입니다. 메인 Claude의 초안을 비판적으로 검토합니다."
2. 주제 및 맥락 설명
3. 현재 라운드 번호와 모드
4. 검토 대상 내용 (초안/수정안 전문)
5. 이전 라운드 피드백 (2라운드 이상일 경우)
6. 모드별 톤 지시 (건설적 피드백 / 비판적 반론 등)
7. 웹검색 지시 (주제 유형에 따라)
8. 응답 형식 지시
```

### 독립성 보장

서브에이전트가 메인 Claude의 결론을 그대로 추종하지 않도록 다음 문구를 프롬프트에 포함:

> "메인 Claude와 독립적인 시각에서 검토하라. 동의하지 않는 부분은 명확히 반박하고, 누락된 관점이나 대안을 제시하라."

## 세션 관리

### 단일 라운드 (1 Round, 2 Round)

각 서브에이전트 호출은 독립적이다. 이전 맥락이 필요한 경우 프롬프트에 이전 라운드의 내용을 직접 포함한다.

```
Agent({
  description: "Round 2 - 수정안 재검토",
  model: "sonnet",
  prompt: "... [이전 초안 내용] ... [이전 검토 내용] ... [수정된 내용] ... 재검토해주세요."
})
```

### 다중 라운드 (Adaptive, Devil's Advocate)

`SendMessage`로 기존 서브에이전트 세션을 이어갈 수 있다. 동일 서브에이전트에 추가 메시지를 보내면 이전 맥락이 유지된다.

```
# 첫 호출
Agent({
  description: "Round 1 - 검토",
  model: "sonnet",
  prompt: "..."
})
→ agent_id 확인

# 이후 라운드
SendMessage({
  to: "{agent_id}",
  message: "다음 라운드 내용: ..."
})
```

**주의**: SendMessage 사용이 불가능하거나 맥락이 너무 길 경우, 새 Agent 호출로 대체하되 이전 맥락을 프롬프트에 요약하여 포함한다.

## 응답 처리

Agent 도구의 반환값은 서브에이전트의 응답 텍스트 그 자체이다.

### 메인 Claude의 후처리

1. 서브에이전트 응답을 해당 라운드의 파일로 저장 (예: `round1_2_reviewer_review.md`)
2. 응답 내용을 분석하여 판단(decision) 작성
3. 다음 라운드 진행 여부 결정

### 서브에이전트 호출 실패 시

Agent 도구 호출이 실패한 경우:
1. 1회 재시도
2. 재시도 실패 시 메인 Claude가 단독으로 해당 라운드를 수행
3. `collab_summary.md`에 실패 사실 기록
